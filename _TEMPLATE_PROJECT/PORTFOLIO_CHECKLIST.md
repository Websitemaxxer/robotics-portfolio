# Portfolio Evidence Checklist — [PROJECT NAME]

Tick each item once the evidence actually exists in the repo. The goal is that
someone can open this project and find proof of every claim. Each line says
**where** that evidence should live.

## Core evidence

- [ ] **Problem statement** — who it's for, the problem, measurable success criteria → `01_planning/PROBLEM_STATEMENT.md`
- [ ] **System diagram** — a sketch of how the parts connect (photo of a hand drawing is fine) → `05_media/photos/` (linked from the problem statement)
- [ ] **Build plan** — week-by-week plan → `01_planning/BUILD_PLAN.md`
- [ ] **CAD / version history** — model files or exported drawings; note revisions → `02_cad/`
- [ ] **BOM with costs** — every part, quantity and price → `03_electronics/BOM.csv`
- [ ] **Wiring diagram + pin map** → `03_electronics/WIRING.md`
- [ ] **Code in the repo** — the actual sketches/firmware, not screenshots → `04_code/`
- [ ] **Dated build diary with photos** — an entry per session → `01_planning/BUILD_DIARY.md`
- [ ] **At least one documented failure + fix** — what broke and how I solved it → `01_planning/BUILD_DIARY.md`
- [ ] **Quantitative test results** — numbers vs targets, pass/fail → `06_tests/TEST_LOG.md`
- [ ] **Demo video** — short clip of it working → `05_media/videos/`
- [ ] **Reflection** — what was original vs adapted, what I'd change → `07_reflection/REFLECTION.md`

## Polish (do this before calling it "done")

- [ ] `README.md` filled in with a **hero photo** and a **results table** that has real numbers
- [ ] A **row for this project added to the top-level `README.md`** Projects table
- [ ] The **Skills checklist in the top-level `README.md`** updated to tick anything this project now proves
